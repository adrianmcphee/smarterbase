Supported SQL

Single-table CRUD covers 90% of prototype needs

Supported

-- Data Definition
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid7(),
    email TEXT UNIQUE,
    name TEXT
)
CREATE INDEX idx_role ON users(role)

-- Queries
SELECT * FROM users WHERE id = $1
SELECT * FROM users WHERE email = $1
SELECT * FROM users WHERE role = 'admin'
    ORDER BY id DESC LIMIT 10

-- Mutations
INSERT INTO users (email, name) VALUES ($1, $2)
UPDATE users SET name = $1 WHERE id = $2
DELETE FROM users WHERE id = $1

Not Supported (Use PostgreSQL)

-- No JOINs
SELECT * FROM users u
JOIN orders o ON o.user_id = u.id

-- No aggregations
SELECT COUNT(*) FROM users
WHERE role = 'admin'

-- No subqueries
SELECT * FROM users
WHERE id IN (SELECT user_id FROM ...)

-- No transactions
BEGIN; INSERT ...; INSERT ...; COMMIT;

If you need these, you need PostgreSQL. That's the point: migrate when you're ready.

UUIDv7 Primary Keys

Time-ordered UUIDs that sort chronologically

  • Time-ordered - IDs sort chronologically. No need for created_at index.
  • No coordination - Generate IDs anywhere without a central authority.
  • PostgreSQL-native - PostgreSQL 17+ supports UUIDv7. Migration is seamless.
  • Filesystem-friendly - Lexicographic sort = chronological sort. 
019363e8-7a6b-7def-8000-1a2b3c4d5e6f
|        |    |
|        |    +-- version 7 marker
|        +-- random bits
+-- millisecond timestamp (48 bits)

# ORDER BY id DESC = most recent first
# ls ./data/users/ = creation order

When to Use SmarterBase

Use It For

Early development - Explore your data model without migration debt
Prototypes & demos - Self-contained, no database setup required
Single-server production - The pattern works. NVMe is fast.
Learning - See your data as JSON files

Graduate to PostgreSQL When You Need

Transactions - ACID guarantees across operations
JOINs and aggregations - Complex queries
Multi-server - Replication, clustering
Scale - More than ~1M rows/table

Export to PostgreSQL anytime. Your queries already work.

Quick Start

Get started in minutes

1

Install & Configure LLM

# Install
go install github.com/adrianmcphee/smarterbase/cmd/smarterbase@latest

# Configure your LLM provider (pick one)
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
export GEMINI_API_KEY="..."
2

Design Your Schema with AI

smarterbase design
> Describe your project: E-commerce marketplace
> AI suggests tables: users, products, orders, reviews...
> Refine until it feels right
3

Generate Realistic Data & Start Server

# Generate realistic synthetic data
smarterbase generate --count 100

# Start the server
smarterbase serve --port 5433 --data ./data
4

Connect from Any Language

# Python / SQLAlchemy / Django
DATABASE_URL = "postgresql://localhost:5433/myapp"

# Node.js / Prisma / TypeORM
const pool = new Pool({ host: 'localhost', port: 5433 });

# Ruby / Rails / ActiveRecord
host: localhost
port: 5433

# Go / GORM / sqlx
db, _ := sql.Open("postgres", "host=localhost port=5433")
5

Migrate to PostgreSQL (When Ready)

# Export your data
smarterbase export > dump.sql

# Import to PostgreSQL
psql myapp < dump.sql

# Update your config
DATABASE_URL = "postgresql://prod-db:5432/myapp"
